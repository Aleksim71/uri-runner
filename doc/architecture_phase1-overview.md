# Архитектурная модель

В Phase 1 система разделена на три основных слоя.

## CLI Layer

CLI отвечает за запуск команд пользователя.

Примеры:

    uri test watcher
    uri doctor
    uri run RUNBOOK.yaml

CLI не содержит логики выполнения и только передаёт управление runtime.

------------------------------------------------------------------------

## URAM Runtime

URAM является runtime-движком URI.

Он отвечает за:

-   загрузку сценария
-   управление выполнением
-   хранение состояния выполнения
-   вызов команд через registry

Основные компоненты runtime:

    scenario reader
    scenario parser
    scenario executor
    command resolver
    command registry

------------------------------------------------------------------------

## Commands Layer

Команды реализуют атомарные действия системы.

Команды сгруппированы в библиотеки:

    context
    file
    test
    system

Команды вызываются runtime-движком через registry.

------------------------------------------------------------------------

# Execution Model

URI выполняет **сценарий**, который хранится в теле runbook или
context-файла.

Сценарий представляет собой последовательность шагов.

Каждый шаг:

-   вызывает команду
-   получает результат выполнения
-   может изменить поток выполнения

Execution flow:

    CLI
    ↓
    URAM runtime
    ↓
    Scenario reader
    ↓
    Scenario executor
    ↓
    Command registry
    ↓
    Command handler

------------------------------------------------------------------------

# Сценарии выполнения

Сценарии описываются в теле файла и определяют:

-   какие команды выполняются
-   в каком порядке
-   какие условия влияют на переход между шагами
