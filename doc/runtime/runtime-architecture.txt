URI Runner
Архитектура Runtime

Версия: 2 (A13)


Обзор

Архитектура Runtime описывает execution-ядро URI Runner.

URI — artifact-driven execution engine.


Ключевое изменение (A13)

Execution теперь основан на materialized plan.

Runbook больше не исполняется напрямую.


Pipeline выполнения (A13)

RUNBOOK
→ compile
→ materialize plan
→ plan validation
→ run-plan
→ execution
→ trace / history / outbox


Совместимость

materializePlanFromRunbook:
- используется watcher и legacy flows
- компилирует runbook через compileRunbookFile
- сохраняет plan.json

materializePlan:
- новый API
- возвращает plan объект
- используется adapter-логикой


Система артефактов

Основные execution-артефакты:

PLAN (plan.json)  
TRACE  
RESULT  
META  


Гарантии

- каждый run создаёт plan artifact до выполнения  
- execution всегда идёт через plan  
- history содержит planRelPath  


Взаимодействие с AI

URI взаимодействует с AI через inbox/outbox protocol.
