Протокол Inbox–Outbox


Выходы URI делятся на две категории.


Внешний выход (для AI)

outbox.zip

Содержимое:

outbox.json
предоставленные данные


Внутренний выход (runtime history)

trace.json
events.jsonl


Важное правило

Внешний AI получает только outbox.zip.

Внутренние execution-traces остаются внутри runtime history.


Назначение

outbox.json → коммуникация с AI
trace.json → отладка и replay


Получатель (receiver)

Каждый входящий пакет должен содержать поле:

receiver

Пример:

receiver: uri

Назначение поля receiver:

определяет, какому агенту адресован входящий пакет.

Правило:

URI обрабатывает только пакеты, где:

receiver == uri

Пакеты с другим receiver:
- игнорируются
- не попадают в inbox
- не создают outbox
- не считаются ошибкой

Важно:

Внешний AI (Макс) не получает информацию
о проигнорированных пакетах.
