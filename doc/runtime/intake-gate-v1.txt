URI Runner
Intake Gate v1

Назначение

Статус

Граница ответственности подтверждена в A16.


Intake Gate отвечает только за отбор входящих пакетов до их попадания в Inbox.

Это не слой проверки полноты обязательных файлов проекта.

Алгоритм

1. Обнаружение пакета в папке загрузки
2. Открытие пакета без полного распаковывания
3. Чтение RUNBOOK.yaml
4. Извлечение поля receiver
5. Принятие решения

Решения

ACCEPT:
receiver == uri
→ пакет переносится в Inbox

IGNORE:
receiver != uri
→ пакет игнорируется

REJECT:
receiver отсутствует или metadata некорректна
→ пакет отклоняется

Свойства

- Intake Gate не создаёт trace
- Intake Gate не создаёт outbox
- Intake Gate не запускает сценарий
- Intake Gate не пишет логи в v1
- Intake Gate не проверяет, существуют ли требуемые проектные файлы для handoff Максу

Граница

Execution scenario начинается только после ACCEPT.

Граница ответственности

Новый слой Required Files Delivery Report V1 находится НЕ здесь.

Он относится к handoff / surface path и начинается только после того,
как пакет уже принят и основной execution-путь завершён.

Правильное разделение:

Intake Gate:
- принимает или отклоняет входящий пакет

Runtime / Handoff layer:
- собирает requested files
- классифицирует `provided / missing / failed`
- формирует delivery report
- при необходимости прикладывает дерево проекта

Следствие

Проверку полноты обязательных файлов нельзя смешивать с `receiver`-gate.
Иначе watcher-уровень начнёт выполнять чужую ответственность.
