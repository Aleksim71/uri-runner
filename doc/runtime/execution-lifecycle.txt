Materialize Plan

RUNBOOK преобразуется в plan:

- нормализация  
- валидация  
- сохранение plan.json  

Каждый run создаёт plan до выполнения.


Plan Validation (A14)

На этом этапе runtime проверяет структурный контракт plan:

- plan является объектом  
- присутствует plan.version  
- plan.steps является массивом  
- каждый шаг (step) корректной структуры  

Если validation не пройдена:

- execution не начинается  
- возвращается ошибка PLAN_SCHEMA_INVALID  


Execution

Выполнение происходит через run-plan.

Runtime не исполняет runbook напрямую.


History

History сохраняет:

- trace  
- outbox  
- planRelPath  

planRelPath указывает на plan.json.
