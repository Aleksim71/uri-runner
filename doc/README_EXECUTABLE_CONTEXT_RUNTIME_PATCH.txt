Executable Context Runtime Patch
URI Runner Next

------------------------------------------------
Идея
------------------------------------------------

Добавляется новый механизм исполнения:

contexts/system/executable.yaml

Это единственный исполняемый контекст
для проекта.

AI может переписывать этот файл.

URI читает его
и выполняет.

------------------------------------------------
Зачем это нужно
------------------------------------------------

RUNBOOK.yaml из inbox.zip
не хранит историю задачи.

Git хранит историю изменения
executable.yaml.

Поэтому:

контекст + команда
хранятся вместе.

------------------------------------------------
Структура
------------------------------------------------

contexts/
  system/
    executable.yaml

------------------------------------------------
Пример executable.yaml
------------------------------------------------

version: 1

meta:
  context_kind: executable_context
  context_id: system_executable
  project: uri-runner-next
  status: active

scenario:
  start: step1

steps:
  - id: step1
    command: system.echo
    args:
      message: hello
    on_success: step2

  - id: step2
    command: system.echo
    args:
      message: done
    stop: true

------------------------------------------------
Lifecycle
------------------------------------------------

AI пишет executable.yaml

URI:

1 читает
2 исполняет
3 очищает тело

------------------------------------------------
После исполнения
------------------------------------------------

version: 1

meta:
  context_kind: executable_context
  context_id: system_executable
  status: executed
  last_run_id: RUN_ID
  last_run_ok: true
  last_engine: scenario

------------------------------------------------
Почему только один файл
------------------------------------------------

если каждый запуск создавать новый
контекстный файл

папка contexts разрастётся
до миллионов файлов.

поэтому:

1 executable файл
+ git history.

------------------------------------------------
Безопасность
------------------------------------------------

AI не имеет доступа
к файловой системе.

URI:

сам определяет

cwd
paths
workspace

------------------------------------------------
Следующий шаг
------------------------------------------------

URI сможет запускать
executable.yaml

даже если inbox.zip отсутствует.

команда:

uri run

будет проверять:

contexts/system/executable.yaml

и выполнять его напрямую.
