TASKLIST_STAGE3.txt

PROJECT
URI Runner

STAGE
Stage 3 — Execution Safety (A8–A12)

GOAL
Implement controlled execution safety for uri run-plan:
- environment reset before execution
- per-step validation after each command
- final goal validation after all commands
- git checkpoint before changes
- rollback tracked files on failure
- database backup/restore on failure
- unified report inside outbox.zip

────────────────────────────────

Wave 1

A9 Step Checks
A10 Goal Checks

Wave 2
A11 Git Checkpoint + Rollback

Wave 3
A12 DB Backup/Restore + Unified Report

────────────────────────────────
MINIMAL POST-STAGE SMOKE FLOW
────────────────────────────────

Expected flow after Stage 3 implementation:

uri compile inbox.zip runtime/test-plan.json
uri run-plan runtime/test-plan.json

Internal sequence:
- create git checkpoint
- create DB backup if enabled
- stop managed processes
- cleanup runtime state
- start server
- run healthcheck
- execute step 1
- run step 1 checks
- execute remaining steps
- run goal checks
- on failure run rollback
- write trace
- update history index
- write outbox.zip report


────────────────────────────────
NON-GOALS FOR THIS STAGE
────────────────────────────────

Do not include yet:
- parallel execution
- graph execution
- multi-root orchestration
- external HTTP side-effect rollback
- full workspace clone engine
- advanced DB snapshot engines beyond dump/restore

SUCCESS CRITERIA FOR WHOLE STAGE

- each step can be validated locally
- final goal state is validated explicitly
- failure triggers tracked-file rollback
- optional DB restore works through dump/restore
- outbox.zip contains a full execution report
